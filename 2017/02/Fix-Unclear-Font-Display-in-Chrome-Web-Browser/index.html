修复 Linux 英文语境下 Chrome 内中文字体问题

Posted on 14 February 2017 | In Programming , Front-end Development |

Preface

刚开始用 Linux Mint 的时候 系统语言是设置成中文的，带来的问题就是 想在 tty 里输入中文简直吃屎，甚至连中文都无法显示出来，我要是再装个什么软件包来解决这个问题 没准又要把我装好的系统搞坏掉。
于是我干脆在安装系统时候直接把语言设置为英文（en_US）。然后问题又来了，KDE Dolphin 里显示中文文件名的时候，那文字东倒西歪的 有些字甚至还在用不知道什么时候的旧写法。其实这个还好，麻烦更多的还是在浏览器这里。
FireFox 和 Chrome 都在使用 “远古字体”，好在 FireFox 渲染出来得效果还行 至少不是东倒西歪那样的。Chrome 就不行了，它和 Dolphin 一样东倒西歪。虽然可以把 Chrome 设置里的字体设定为中文字体，但会导致原有的一些英文字符被代替为用中文的 Full Width Character。一个典型的例子就是双引号，变成中文的双引号后 引号正反对不上。Chrome Web Store 里虽然有 Advanced Font Settings 这个扩展，但也无济于事。
最后只能借助另一款 Chrome 扩展：Stylish，这个扩展可以在任意网页嵌入自定义 CSS 来达到修改页面样式的目的。借助它 我可以编写自己的 CSS，这样就能修改网页内的字体设置了。
